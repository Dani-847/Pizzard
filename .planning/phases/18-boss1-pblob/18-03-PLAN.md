# Plan 18.3: P'blob Phase 3 (Grid Puzzle & Death)

## Objective
Implement the final Phase 3 grid puzzle and wire the death screen loop for Boss 1.

## Mechanics
At 33% HP:
1. Boss becomes invulnerable, moves to Top Center.
2. Player is teleported to Bottom Center.
3. A grid of tiles spawns between them.
4. Animation: Grid starts gray -> flashes Red/Green -> reveals Green walkable path.
5. Player walks the path. Red tiles slow player and deal 1 DMG/sec.
6. Reaching the end (next to boss) triggers final combat phase (boss permanently vulnerable until 0 HP).
7. Upon death (player), verify DeathUI returns to Shop preserving tokens, or Main Menu.

## Tasks
1. **Grid Generation (`PblobGridPuzzle.cs`):**
   - Build a script to spawn a 2D array of Tile prefabs.
   - Generate a continuous path from bottom to top (drunkard's walk or simple DFS finding a path). The path nodes are "Green", all others are "Red".
   - `Coroutine AnimateGrid()`: Turn all Gray -> rapid flash Red/Green (0.5s intervals) -> settle with Path=Green, Rest=Red.
   - After 2 seconds, flip all tiles back to Gray (or keep them colored? "reveals the green route ... if he steps in red" suggests they might stay colored or turn invisible, we will keep them visible for clarity unless specified otherwise. We will assume the colors stay visible to guide the player). 
   
2. **Tile Logic:**
   - Tile uses `BoxCollider2D` Trigger.
   - If player enters Red tile: apply Slow debuff (reduce MoveSpeed), start Coroutine to deal 1 DMG / sec.
   - `OnTriggerExit2D`: remove Slow debuff, stop DMG coroutine.

3. **Phase 3 Logic (`PblobController.cs`):**
   - `Phase3Transition`: Move boss to Top Center. Teleport Player to Bottom Center. Create Grid. Lock boss behind an invisible wall if necessary to force the path.
   - `Phase3_Grid`: Wait for player to enter a trigger zone near the boss.
   - `Phase3_Combat`: Destroy/disable Grid. Boss becomes `MakeVulnerable()` permanently. Boss attacks wildly (Speed up `PblobAttackPattern1` significantly).
   - If HP hits 0: Trigger `Defeat()`.

4. **Death Screen Verification:**
   - The user specified: "when dying in a boss player gets 2 buttons: looping back to last shop (without spending original tokens) or main menu".
   - Verify `DeathUI.cs` `OnReintentar()` uses the save file checkpoint correctly. Since Phase 15 integrated `GameFlowManager.Instance.ChangeState(GameState.Shop)` and `SaveSystem`, we just need to ensure the Token count loaded matches the state *before* the boss fight. (If Shop was the last save point, tokens spent *in* the shop might be lost if we revert to *before* the shop. We will flag this for testing).
