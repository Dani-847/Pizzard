# Plan 18.5: Boss Fixes - Moustache Points, Grid, and Damage

## Objective
Fix the 4 specific issues reported by the user regarding Phase 1 moustache points movement, Phase 2 timeout damage, Phase 3 grid shape/collisions, and Phase 3 Boss contact vulnerability.

## Bugs & Root Causes
1. **Phase 1: Moustache points are children of the boss.**
   - *Cause:* `mustachePoints` are child transforms, so they move when the boss moves.
   - *Fix:* In `PblobAttackPattern1.cs`, iterate over `mustachePoints` during `Start()` and call `point.SetParent(null)`. This leaves them at their original world positions as fixed parts of the arena.

2. **Phase 2: After timers, player HP remains the same.**
   - *Cause:* The logic calls `playerHP.RecibirDaĂ±o(penaltyDamage)` but we need to verify `penaltyDamage` is > 0 and the player is actually receiving the hit (no invulnerability frames blocking it, etc.).
   - *Fix:* Ensure `penaltyDamage` calculates properly. Add `Debug.Log` inside the penalty block to trace why it might be failing. (Will verify if `PlayerHPController` requires a specific layer or if invulnerability is blocking).

3. **Phase 3: Make the grid more compact (5 x 20).**
   - *Cause:* `PblobGridPuzzle.cs` is currently hardcoded to 10x3 with tileSize 1.85.
   - *Fix:* Change `gridWidth = 20`, `gridHeight = 5`, and set `tileSize = 1.0f` to fit exactly in the same ~20-unit wide arena space.

4. **Phase 3: Red squares do nothing when stepped on.**
   - *Cause:* `PblobGridTile` relies on `OnTriggerEnter2D` but the prefab might lack a trigger collider.
   - *Fix:* In `PblobGridTile.Awake()`, dynamically ensure a `BoxCollider2D` exists and `isTrigger = true`.

5. **Phase 3: Boss physical contact does not make boss vulnerable.**
   - *Cause:* `PblobController` is using `OnTriggerEnter2D` but the Boss's collider is solid (not a trigger). Solid colliders fire `OnCollisionEnter2D`.
   - *Fix:* Replace `OnTriggerEnter2D`/`Exit2D` with `OnCollisionEnter2D`/`Exit2D` in `PblobController.cs` for Phase 3 contact vulnerability.
