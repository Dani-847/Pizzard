# Plan 8.1: UI Transition Fixes

## Objective
Fix the two user-reported bugs:
1. `CombinationsUI` overlapping with `OptionsUI`.
2. `DialogUI` failing to progress and displaying as empty.

## Context
During the refactoring of Phase 7, the `GameFlowManager` was set up to directly toggle the root canvases of the UI objects. 
- For `OptionsUI`, `OnClickCombinations` simply toggles the combinations panel on without toggling itself off, causing an overlap.
- For `GameFlowManager`, it is currently calling `ui.dialogUI.Show()` and `ui.tiendaUI.Show()`. These zero-parameter functions bypass the data initialization (`ShowIntroDialog(this)`, `Show(this)`) meaning the dialog has no text to display, and the shop has no reference to the FlowManager to allow advancing to the next phase.

## Proposed Changes

### GameFlowManager Integration
#### [MODIFY] `Assets/Scripts/GameFlowManager/GameFlowManager.cs`
Update the `ChangeState` method's switch statement to invoke the specifically targeted setup methods:
- Change `ui.dialogUI.Show();` to `ui.dialogUI.ShowIntroDialog(this);` for `GameState.IntroDialog`.
- Change `ui.dialogUI.Show();` to `ui.dialogUI.ShowPreBossDialog(this);` for `GameState.PreBossDialog`.
- Change `ui.tiendaUI.Show();` to `ui.tiendaUI.Show(this);` for `GameState.Shop`.

### Options UI Overlap
#### [MODIFY] `Assets/Scripts/UI/OptionsUI.cs`
Update `OnClickCombinations()` to call `Hide();` immediately after showing the `CombinationsUI` panel. The `CombinationsUI.cs` already possesses a return function that will cleanly restore the `OptionsUI` when closed!

## Verification Plan

### Manual Verification
- The user will press "Play" in `MainMenu.unity`, click "Opciones", click "Combinaciones" and visually verify the overlap is gone.
- The user will then click "Jugar" and verify that the Introduction Dialog possesses actual text, and that clicking "Next" progresses through the text arrays and successfully transitions the game into the Shop phase.
