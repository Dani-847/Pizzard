**ShopUI.cs changes:**

1. Add a private field for the token source:
```csharp
private ITokenSource _tokenSource;
```

2. Add a public setter method:
```csharp
public void SetTokenSource(ITokenSource source)
{
    _tokenSource = source;
}
```

3. In RefreshTokens() (or wherever BossCurrency is read), change:
```csharp
// OLD:
int tokens = ProgressionManager.Instance.BossCurrency;
// NEW:
ITokenSource src = _tokenSource ?? ProgressionManager.Instance as ITokenSource;
int tokens = src?.GetTokens() ?? 0;
```
This fallback (`?? ProgressionManager.Instance`) ensures the main-game shop continues working without any changes to existing scenes — SetTokenSource only needs to be called from Playground context.

4. Add suppressSave parameter to Hide():
```csharp
// Change signature from:
public void Hide()
// To:
public void Hide(bool suppressSave = false)
```
Inside Hide(), find the SaveManager.SaveGame() call and guard it:
```csharp
if (!suppressSave)
    SaveManager.Instance?.SaveGame();
```

**ShopController.cs changes:**

Find all calls to ProgressionManager.Instance.SpendCurrency(amount) in purchase methods. Replace with:

ITokenSource src = (_tokenSource ?? ProgressionManager.Instance as ITokenSource);

Where _tokenSource should be passed from ShopUI or ShopController needs its own SetTokenSource. Check if ShopController already holds a reference to ShopUI — if so, route through ShopUI's token source. If ShopController is independent, add the same `private ITokenSource _tokenSource` and `SetTokenSource()` pattern.

Critical constraint: Do NOT remove or change any existing calls that work in the main game. The fallback to ProgressionManager.Instance must always be present so existing scenes require zero changes.
