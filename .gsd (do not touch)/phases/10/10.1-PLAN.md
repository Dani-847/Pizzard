# Phase 10: Codebase Refactor, Architecture Standardization & Polish

## Context & Objectives
The core gameplay loop is functional but suffers from architectural debt. Issues like the recent "frozen game" bugs and camera zoom problems stem from inconsistencies in how scenes are built (e.g., duplicated UIs, hardcoded cameras, conflicting physics components). 
This phase will "start over" the scene building process to standardize them, add quality-of-life combat enhancements (boss wandering, camera fixes), and finalize the Shop Token economy requested by the user.

## Proposed Changes

### 1. Codebase Audit & Architecture Standardization
- [ ] **Full Script & Asset Scan:** Audit every script in the project. Document its true purpose, polish it if needed, archive it if unused, and ensure it functions perfectly in isolation before integrating it into the pristine scenes.
- [ ] **Clean Scene Building:** Tear down `BossArena_1`, `Shop`, and `MainMenu`. Rebuild them cleanly with only the minimal required GameObjects. Remove all hardcoded duplicate UIs since `GameFlowManager` injects them dynamically.

### 2. Camera & Viewport Fixes
- [ ] **Cinemachine Confiner Setup:** Implement a wide-view Cinemachine Virtual Camera in the Boss Arena. Configure a deadzone so the camera stays largely still but subtly pans when the player approaches the screen edges, clearly defining the arena boundaries.

### 3. Boss Artificial Intelligence Polish
- [ ] **Dynamic & Random Movement:** Update `PblobController.cs` to add natural drifting/scurrying between attack cooldowns to keep the player moving.
- [ ] **Competitive Attack Spawns:** Ensure boss attacks spawn in logical, challenging positions relative to the player to increase combat depth.
- [ ] **Dramatic Phase Transitions:** Add a physical "knockback" or backward retreat to the boss when the final hit of a phase lands to make transitions feel visceral.

### 4. Economy, Progression & New Shop Mechanics
- [ ] **Element Selection Menu:** Build a new UI panel in the Shop where the player *selects* which specific elements they can use in battle.
- [ ] **Wand Tier Element Limits:** Connect the Wand Tier system to the Selection Menu (e.g., Tier 1 allows selecting 1 element type, Tier 2 allows 2, etc.). Include a "Reset Choices" button.
- [ ] **Shop Tokens Polish:** Verify Token drop logic from Boss defeats and ensure the Shop properly deducts tokens logically and visually.
