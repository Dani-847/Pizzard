## Architectural Improvements
- `ElementSelectionUI.cs` has been decoupled from the real-time avatar instantiation cycle. Instead of relying on a physical `PlayerEquip` mono-script being present in the scene, the UI can now directly access the `EquipableObject` state saved deep in the active `SaveManager`. This allows Elements to be mapped in menus (like the Shop or pre-game lobby) securely without breaking memory states.
- `PersistentEventSystem.cs` no longer passively hopes scenes don't contain other `EventSystem` objects. It hooks onto `SceneManager.sceneLoaded` specifically to seek and destroy intruding UI Event modules explicitly, permanently ending the "Multiple EventSystems" conflict.
