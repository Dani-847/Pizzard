<objective>
Simplify and harden the save system for v1.0. Only one save behavior matters:

1. **Death/Retry** → return to Shop with element selection + unspent tokens preserved
2. **Main Menu / Quit** → full reset, everything starts from 0

No disk persistence needed beyond language setting. No "Continue" button. No corruption handling.

Current state analysis:
- `VolverATiendaTrasMuerte()` calls `LoadGame()` which reads from disk — this is overly complex. The retry state should just be kept in memory.
- `VolverAlMenu()` only changes state but does NOT call `ResetSave()` — tokens/elements leak.
- `pendingWandElements` is `[NonSerialized]` so elements don't persist across retries correctly.
- `IniciarJuego()` calls `ResetSave()` correctly but also grants starting tokens — this is fine.
</objective>

In `GameFlowManager.VolverATiendaTrasMuerte()`:

The current flow calls `LoadGame()` which reads from disk. This is fragile (disk state may be stale). Instead:
1. Do NOT call `LoadGame()`. The in-memory state already has the correct element selection and tokens.
2. Simply change state to Shop. The tokens (via `ProgressionManager.BossCurrency`) are already in memory.
3. The `pendingWandElements` list (in SaveData) should already contain the player's element palette from when they entered combat.

Current code:
```csharp
public void VolverATiendaTrasMuerte()
{
    Time.timeScale = 1f;
    if (Progression.SaveManager.Instance != null && HasSavedGame())
    {
        Debug.Log("[GameFlowManager] Player died. Reloading last save...");
        Progression.SaveManager.Instance.LoadGame();
    }
    ChangeState(GameState.Shop);
    StartCoroutine(ShowDeathDialogAfterLoad());
}
```

Should become:
```csharp
public void VolverATiendaTrasMuerte()
{
    Time.timeScale = 1f;
    // Don't reload from disk — in-memory state already has correct elements + tokens.
    // Just go back to Shop with current state preserved.
    Debug.Log("[GameFlowManager] Player died. Returning to shop with current elements + tokens.");
    ChangeState(GameState.Shop);
    StartCoroutine(ShowDeathDialogAfterLoad());
}
```

VERIFY: After this change, tokens and element selection should survive death/retry. The player keeps what they had when they entered the boss fight.
