# Plan 18.4: P'blob Bug Fixes (Health Bar, Phase 3, Pause, Debug Cleanup)

## Objective
Fix all remaining P'blob bugs discovered during playtesting: health bar not reacting to damage, Phase 3 grid not spawning, Phase 3 boss resuming Phase 1 behaviour, inability to pause during boss fight, and leftover debug keys.

## Bugs
1. **Health bar not draining** — `DelayedHealthBar` rebuilds bars but old children (`BackHealthBar`, `HealthBackBar` etc.) accumulate as duplicates; `fillAmount` never updates because `SetHealth()` is called before `Start()` finishes building the fresh bars.
2. **Phase 3 grid missing** — `gridPuzzle` field on `PblobController` is unassigned in the Pblob prefab → `GenerateGrid()` never fires.
3. **Phase 3 boss repeats Phase 1** — `Phase3_Combat` in `ChangeState()` calls `attackPatterns[0].StartPattern()` — identical to Phase 1.
4. **No Esc pause during boss fight** — No pause input handler exists in the boss arena scene.
5. **Debug keys still active** — `Update()` still reads `T` (damage) and `P` (next phase) when `debugMode = true`.

## Tasks

1. **Rebuild `DelayedHealthBar.cs` — reliable, duplicate-free bars:**
   - `ClearAllChildrenExceptBorder()`: destroy every child Transform that is NOT named `PblobBorderHealthBar`.
   - `BuildBars()`: guard with `if (frontImg != null) return;` — then create `HealthBackBar` (orange) and `HealthFrontBar` (red) as new GameObjects with `Image.type = Filled`, `FillMethod.Horizontal`, `fillAmount = 1f`, `sprite = null`. Mirror identical `anchorMin/Max/offsetMin/offsetMax` from `PblobBorderHealthBar`. Push border to last sibling.
   - `SetHealth(float ratio)`: instantly set `frontImg.fillAmount = ratio`. Start `DrainBack()` coroutine that waits `GameBalance.Bosses.Pblob.HealthBarDrainDelay` then `MoveTowards` `backImg.fillAmount` to ratio at `HealthBarDrainSpeed` per second.
   - Remove the `UpdateHealthBar(1f)` call from `PblobUI.Start()` — bars already initialise to full inside `BuildBars()`.
   - Add `HealthBarDrainDelay = 0.5f`, `HealthBarDrainSpeed = 0.35f` to `GameBalance.Bosses.Pblob`.

2. **Fix Phase 3 grid + player relocation (`PblobController.cs`):**
   - In `Start()`: add `if (gridPuzzle == null) gridPuzzle = FindObjectOfType<PblobGridPuzzle>();`
   - In `ChangeState(Phase3_Grid)`: use `arenaCenter + new Vector3(0, GameBalance.Bosses.Pblob.GridSpawnOffsetY, 0)` as spawn position.
   - `Phase3TransitionRoutine()`: player target = `arenaCenter + (0, GridSpawnOffsetY + 1f, 0)`; boss target = `arenaCenter + (0, Mathf.Abs(GridSpawnOffsetY) - 2f, 0)`. After Lerp, briefly disable `PlayerMovement` for 2s so player doesn't run off during the grid reveal.
   - Add `GridSpawnOffsetY = -5f`, `Phase3MoveSpeed = 6f` to `GameBalance.Bosses.Pblob`.

3. **Phase 3 Combat AI — aggressive chase (`PblobController.cs`):**
   - Add `Phase3CombatRoutine()` coroutine: `MakeVulnerable()`, then loop `MoveTowards(playerTransform.position, Phase3MoveSpeed * Time.deltaTime)` while `currentState == Phase3_Combat`. If `attackPatterns.Length > 1` use `attackPatterns[1]`; otherwise pure chase.
   - Replace `Phase3_Combat` case in `ChangeState()` — remove `attackPatterns[0].StartPattern()`, instead start `Phase3CombatRoutine()`.

4. **Esc pause + debug key removal (`PblobController.cs` + new script):**
   - Delete `[Header("DEBUG")]` fields (`debugMode`, `damageKey`, `nextPhaseKey`), the entire `Update()` method, and `ForceTakeDamage()` / `ForceNextPhase()` helpers.
   - Create `Assets/Scripts/GameFlowManager/PauseBossArena.cs`: `Update()` listens for `KeyCode.Escape` → toggles `isPaused`, sets `Time.timeScale` to 0/1, calls `MenuUI.ShowPauseMenu(isPaused)` (look up actual method on `MenuUI`/`UIManager` before writing).
   - Add `PauseBossArena` component to the `Pblob` prefab.

5. **GameBalance constant cleanup (`GameBalance.cs`):**
   - Replace the hardcoded `* 1.5f` circle scale and `3.5f` spawn radius in `SpawnPhase2Circles()` with `GameBalance.Bosses.Pblob.CircleScale` and `CircleSpawnRadius`.
   - Add: `CircleScale = 1.5f`, `CircleSpawnRadius = 3.5f`, `GridSpawnOffsetY = -5f`, `Phase3MoveSpeed = 6f`, `HealthBarDrainDelay = 0.5f`, `HealthBarDrainSpeed = 0.35f`.

## Verification
- Boss health bar shows red full at start; taking damage immediately shortens red bar; orange bar catches up slowly after ~0.5s.
- PblobUI hierarchy has exactly 3 children: `HealthBackBar`, `HealthFrontBar`, `PblobBorderHealthBar` — no duplicates.
- Boss HP at 33% → grid appears below boss, player teleports to bottom of arena.
- After completing grid path → boss chases player constantly at high speed, does NOT fire Phase 1 projectile patterns.
- Press Esc mid-fight → game pauses. Press Esc again → resumes.
- Pressing T or P in play mode does nothing.
