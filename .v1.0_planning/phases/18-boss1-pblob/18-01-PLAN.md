# Plan 18.1: P'blob Core Refactor & Phase 1

## Objective
Refactor `PblobController` into a robust state-driven machine and implement Phase 1 logic.

## Motivation
The current `PblobController` blindly cycles vulnerability timers. The new design requires an explicit IDLE state, Phase 1 (100% -> 66%), Phase 2 (66% -> 33%), and Phase 3 (<33%). Phase 1 requires specific movement-based vulnerability toggling.

## Tasks
1. **GameBalance Preparation:**
   - Add the following to `GameBalance.Bosses.Pblob`:
     - `Phase1AlternateTime = 2f`
     - `Phase1MoveSpeed = [appropriate value]`
     - `Phase2Timer = 30f`
     - `Phase2CircleMoveTime = 5f`
     - `GridDamagePerTick = 1f` 
   - Clean up outdated variables like `MaxPhases = 3` and `PhaseHPPercent` if no longer used.

2. **State Machine Setup (`PblobController.cs`):**
   - Define an `enum PblobState { Idle, Phase1, Phase2Transition, Phase2, Phase3Transition, Phase3_Grid, Phase3_Combat, Defeated }`.
   - Implement `ChangeState(PblobState newState)`.
   - Ensure the Boss starts in `Idle` in the center of the arena, completely invulnerable.

3. **Phase 1 Implementation:**
   - Override `TakeDamage()`: if hit while `Idle`, transition to `Phase1`.
   - `Phase1` state Coroutine:
     - Loop while state == Phase1:
       - **Move State (2s):** `MakeVulnerable()`, pick random destination near player, move towards it using `GameBalance` speed.
       - **Stand Still State (2s):** `MakeInvulnerable()`, stop moving. Shoot projectiles (using the existing `PblobAttackPattern1` logic).
   - Check HP Threshold: If HP falls to 66% (2/3 of MaxHP), cancel Phase 1 coroutine, trigger `ChangeState(Phase2Transition)`.

## Verification
- Enter `BossArena_1`. Verify boss does nothing until hit.
- Hit boss. Verify it starts moving/stopping in 2s alternating intervals.
- Verify boss can ONLY be damaged while moving.
- Verify boss transitions to a dummy Phase 2 state exactly at 66% HP (e.g. `Debug.Log("Transitioning to Phase 2")`).
