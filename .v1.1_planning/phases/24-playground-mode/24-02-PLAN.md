Add the following to MenuUI.cs following the exact existing button pattern:

```csharp
[Tooltip("Opens Playground scene")]
public Button botonPlayground;
[SerializeField] private Animator playgroundButtonAnimator;
```

2. Add private in-memory flag at class level (NOT serialized, NOT PlayerPrefs):
```csharp
private bool _playgroundPulseActive = true;
```
This resets to true every game launch automatically — no PlayerPrefs needed.

3. In Start(), after existing button wiring, add:
```csharp
if (botonPlayground != null)
    botonPlayground.onClick.AddListener(OnClickPlayground);
if (_playgroundPulseActive && playgroundButtonAnimator != null)
{
    playgroundButtonAnimator.updateMode = AnimatorUpdateMode.UnscaledTime;
    playgroundButtonAnimator.SetBool("Pulsing", true);
}
```
Note: updateMode = UnscaledTime ensures pulse continues even if Time.timeScale is 0.

4. Add new method alongside existing OnClick methods:
```csharp
public void OnClickPlayground()
{
    if (playgroundButtonAnimator != null)
        playgroundButtonAnimator.SetBool("Pulsing", false);
    _playgroundPulseActive = false;
    SceneLoader.LoadScene("PlaygroundScene");
}
```

Do NOT add any PlayerPrefs read/write. Do NOT touch ProgressionManager or SaveManager.

Create a Unity Animator Controller at Assets/Animations/UI/PlaygroundButtonAnimator.controller.

The controller needs two states:
- "Idle" state: default state, no animation clip needed (or an empty clip), scale stays at 1.0
- "Pulsing" state: plays a looping scale pulse animation (1.0 → 1.05 → 1.0)

Create the pulse animation clip at Assets/Animations/UI/PlaygroundButtonPulse.anim:
- Property: RectTransform.localScale
- Keyframe at 0s: (1, 1, 1)
- Keyframe at 0.4s: (1.05, 1.05, 1)
- Keyframe at 0.8s: (1, 1, 1)
- Loop: enabled (wrap mode = Loop)
- Use smooth tangents for organic feel

Animator Controller transitions:
- "Any State" → Pulsing: condition `Pulsing == true`, transition duration 0.1s
- Pulsing → Idle: condition `Pulsing == false`, transition duration 0.1s, has exit time = false

The bool parameter name is "Pulsing" (capital P — must match exactly what MenuUI.cs SetBool uses).

If Assets/Animations/UI/ directory does not exist, create it.

After creating the controller, the Unity inspector must be used to wire playgroundButtonAnimator on the MenuUI GameObject (or the Playground button GameObject) — note this as a scene setup step that happens during scene building in Plan 04.
