# Phase 11 Plan: Critical Loop & UI Fixes

## Objective
Address the critical bugs reported by the user affecting the core gameplay loop, camera tracking, and UI feedback, restoring full playability from Shop to Boss to Loop.

## Reported Issues
1. **Health bar always white**: Player is alive but the UI does not show visual feedback of current health.
2. **Camera doesn't follow main character**: The CameraFollow script isn't tracking properly, possibly due to detachment or mathematical clamping logic.
3. **Shop element selection missing**: The element selection buttons do not appear in the Shop UI.
4. **Boss defeat loop broken**: Killing the boss does not advance the GameFlowManager to the next boss.

## Proposed Implementation Steps

### Task 1: Fix Player Health Bar Feedback
- **Analysis**: Check `CharacterHPUI.cs` and `PlayerHPController.cs`. If the health bar is white, it means the Image component's sprites aren't updating or the UI canvas isn't receiving the events properly.
- **Action**: Fix `CharacterHPUI.ActualizarUI()` to ensure it correctly maps hearts to sprites or colors, and verify the UI prefab references are intact in the `MainMenu` UI instance.

### Task 2: Fix Camera Following
- **Analysis**: Check `CameraFollow.cs`. The limits `minBounds` and `maxBounds` might be overly restrictive, or the `target` parameter is losing the player reference when moving between scenes.
- **Action**: Explicitly ensure `CameraFollow` finds the Player object dynamically on `Update` or `Start` if the reference is lost, and review the bounding mathematics so it doesn't hard-lock the camera.

### Task 3: Fix Shop Element Selection UI
- **Analysis**: Check `ShopUI.cs` and the Shop Canvas hierarchy. The `ElementSelectionUI` fallback was added, but the GameObject might be disabled completely or missing from the prefab.
- **Action**: Examine the `UI` prefab hierarchy via MCP. Ensure the `CombinationsUI` or `ElementSelectionUI` panel is enabled when the Shop state is triggered.

### Task 4: Fix Boss Defeat Progression Loop
- **Analysis**: Check `GameFlowManager.AvanzarFase()` and `PblobController.Defeat()`. When the Boss dies, it must tell the flow manager to move to `PostBossDialog` and then increment `currentBossIndex`.
- **Action**: Ensure `PblobController`'s `OnBossDefeated` UnityEvent is linked properly, or hardcode a fallback that calls `GameFlowManager.Instance.AvanzarFase()` upon boss death.

## Verification
- **Test Health UI**: Take damage in BossArena and verify hearts deplete visually.
- **Test Camera**: Walk to the edges of BossArena and verify the camera pans appropriately.
- **Test Shop**: Enter Shop phase and verify Element buttons are visible and clickable.
- **Test Loop**: Kill the Boss using debug buttons and verify the transition to PostBossDialog and then the next Shop phase.
