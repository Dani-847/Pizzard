# Phase 13 Plan 2: Centralize Game Balance Variables — Summary

**One-liner:** Created `Assets/Scripts/Core/GameBalance.cs` as single source of truth for 123+ balance constants, replacing hardcoded field initializers across 35 scripts.

## What Was Built

A single static class `GameBalance` with nested static classes organized by game subsystem:

```
GameBalance
├── Player        (moveSpeed, dash, HP, invulnerability)
├── Mana          (maxMana, regen, spellCosts dictionary, upgrade multiplier)
├── Potions       (startingMax, healPerPotion)
├── Currency      (bossKillReward)
├── Bosses
│   ├── Pblob     (HP, phases, vulnerability, knockback, wander)
│   ├── Heckiel   (attackInterval, phase2 threshold)
│   ├── Pomodoro  (attackInterval)
│   └── Niggel    (attackInterval, stealRange, stealAmount, speedBuff)
├── Spells
│   ├── Base/Enemy         (speed, lifetime, damage)
│   ├── Pepperoni variants (burn stacks, durations, intervals)
│   ├── Explosion variants (radius, damage, force, fuse)
│   ├── Sticky variants    (stick duration, tick intervals, scaling)
│   ├── PineappleCheese    (absorption, growth, impact)
│   ├── PineapplePepperoni (teleport distance, explosion)
│   ├── Shields            (knockback, reflection, contact damage)
│   └── CheesePepperoniWall (duration, burn, reflection)
├── FireTrail     (charges, interval, duration, radius)
└── Casting       (shield/wall/catapult distances)
```
