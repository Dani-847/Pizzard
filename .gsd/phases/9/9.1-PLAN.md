# Plan 9.1: Core Game Loop, Systems, and Progression Setup

## Objective
Implement the persistent core game loop, token economy, shop limitations, elemental fatigue system, saving/loading, and localization based on user requirements.

## Context
The game currently possesses disconnected prototypes. Phase 9 will unify them into a single continuous loop (Menu -> Dialog -> Shop -> Boss -> Dialog -> Shop) with persistent tokens, a newly defined Elemental Fatigue bar, and an overarching Save System.

## Proposed Changes

### 1. Main Game Loop & Progression
#### [MODIFY] `System/GameFlowManager.cs`
- Enforce the required loop: `MainMenu` -> `IntroDialog` -> `Shop` -> `PreBossDialog` -> `Boss Arena` -> `PostBossDialog` -> `Shop`.
- Trigger Auto-Save when closing the shop and defeating a boss.
- Implement win condition: After Boss 4, show magic pizza and credits.

### 2. Token Currency System & Save System
#### [MODIFY] `Progression/ProgressionManager.cs` (or create `SaveManager.cs`)
- Introduce `Tokens` currency. Boss 1 drops 1 token, others drop 2.
- Implement a JSON-based Save System (editable externally for debug).
- Track: Tokens, Wand Tiers, Max Fatigue, Unlocked Elements, and Boss Progression.

### 3. Raberto's Shop & Wand Upgrades
#### [MODIFY] `UI/ShopUI.cs` & `Player/PlayerEquip.cs`
- Change UI to match new buttons: Upgrade Wand, Upgrade Potions, Increase Fatigue, and the Exit Character Sprite.
- All purchases consume 1 Token. Prevent purchases if Tokens == 0.
- Update `PlayerEquip` to handle 3 Wand Tiers. Tier 1 allows 1 element combinations, Tier 2 allows 2, Tier 3 allows 3+.
- **Exit Logic**: The first shop blocks the player from leaving until they purchase at least the Tier 1 Wand. On subsequent shops, double-clicking the exit triggers a warning dialog first.

### 4. Elemental Fatigue System
#### [NEW] `UI/FatigueUI.cs`
#### [MODIFY] `Player/PlayerController.cs` or `ElementsCombiner.cs`
- Add a new vertical Blue Bar to the left of the screen for 'Elemental Fatigue'.
- Each spell cast consumes fatigue. Recharges slowly over time.
- Shop upgrade increases the maximum capacity (bar drains slower/needs more spells to empty).

### 5. Combat Combination Rules
#### [MODIFY] `ElementsCombiner.cs`
- Apply Tier restrictions derived from the current Wand Tier.
- Implement anti-spam mechanics (detecting identical consecutive casts and applying cooldowns/increased costs).

### 6. Narrative Dialogs & Localization
#### [MODIFY] `UI/DialogUI.cs` & `Core/LocalizationManager.cs`
- Incorporate Bob and Raberto dialog lines for Intro, Pre-Boss, Post-Boss, and Shop Warnings.
- Load `Resources/Languages/en.json` and `es.json` explicitly.
- Allow real-time language toggling from the Options menu.

## Verification Plan

### Automated/Manual Verification
1. Start a New Game and verify the Token counter is 0.
2. Confirm the Shop Exit is blocked on Run 1 until a purchase is made.
3. Validate that casting spells drains the new Fatigue Bar.
4. Defeat Boss 1, ensure 1 Token applies and the JSON Save file is generated.
5. Change language in Options and verify dialogs shift immediately.
